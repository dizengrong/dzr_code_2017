==================== helper系统介绍 ====================

=== 简介
helper是一个以网页形式呈现的，包括两大功能:
	1.查看gm命令列表、物品列表，这个主意是给策划和测试人员使用的
	2.查看在线玩家数据功能，这个主要是给后端开发人员debug用的

=== 如何使用
1.代码依赖于开源库如下几个开源库：
	ranch
	cowlib
	cowboy
	erlydtl
	merl
	依赖于使用rebar和erlang.mk来构建他们
	这些库的代码无法直接嵌入我们的项目代码里面来，因为编译系统的问题
	所以要在linux下使用与项目对应的erl版本来编译代码
	其中，由于erl版本的问题，cowboy只能使用1.0.x的版本
	(本目录下有已经编译好的beam)
2.把上述的已编译好的beam放入一个叫这样的目录中：work_help-1.0.0
  然后放入到服务器安装erlang的lib目录下，比如：/usr/local/lib/erlang/lib
  (如果是本地，可以直接在启动时通过-pa来指定路径就好了)
3.把本文件所在目录的代码放入项目代码库中，然后在world节点启动的地方调用：work_helper_main:start()
4.work_helper是通过一个宏定义来控制是否开启的，具体看启动代码，
  而且如果在erlang的代码库中找不到cowboy库，work_helper也不会启动
